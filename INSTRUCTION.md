# Пошаговая инструкция по разработке сайта Katelkov.by

## Этап 1: Установка библиотек для парсинга данных
1. Установить библиотеку `pip install selenium` - для автоматизации браузера и парсинга динамических страниц
2. Установить библиотеку `pip install beautifulsoup4` - для парсинга HTML-контента
3. Установить библиотеку `pip install requests` - для HTTP-запросов при парсинге
4. Установить библиотеку `pip install lxml` - для быстрого парсинга XML/HTML

## Этап 2: Подключение к базе данных PostgreSQL
5. Установить библиотеку `pip install psycopg2-binary` - драйвер для подключения к PostgreSQL
6. Установить библиотеку `pip install django-environ` - для работы с переменными окружения
7. Создать базу данных PostgreSQL для проекта
8. Настроить подключение к БД в settings.py через переменные окружения

## Этап 3: Настройка безопасности и переменных окружения
9. Установить библиотеку `pip install python-decouple` - альтернатива для работы с .env файлами
10. Создать файл `.env` в папке backend с секретными ключами (SECRET_KEY, DB настройки)
11. Создать файл `.env.example` как шаблон для других разработчиков
12. Обновить settings.py для чтения переменных из .env файла

## Этап 4: Аутентификация и авторизация пользователей
13. Установить библиотеку `pip install djangorestframework-simplejwt` - для JWT токенов
14. Установить библиотеку `pip install django-allauth` - для социальной аутентификации (опционально)
15. Настроить JWT аутентификацию в REST_FRAMEWORK settings
16. Создать приложение `users` для управления пользователями
17. Настроить регистрацию, вход, выход и обновление токенов

## Этап 5: Улучшение API
18. Установить библиотеку `pip install django-filter` - для фильтрации данных в API
19. Установить библиотеку `pip install drf-yasg` - для автоматической документации API (Swagger)
20. Настроить пагинацию для списков товаров
21. Создать сериализаторы для всех моделей
22. Настроить ViewSets и роутеры для REST API

## Этап 6: Модели данных
23. Создать модель `Product` (товар) с полями: название, описание, цена, изображения, характеристики
24. Создать модель `Category` (категория) для группировки товаров
25. Создать модель `Order` (заказ) для хранения заказов пользователей
26. Создать модель `OrderItem` (элемент заказа) для детализации заказов
27. Создать модель `Cart` (корзина) для временного хранения товаров
28. Создать модель `Review` (отзыв) для отзывов о товарах

## Этап 7: Парсер данных о товарах
29. Создать приложение `parser` для парсинга данных
30. Написать скрипт парсера с использованием Selenium и BeautifulSoup
31. Настроить сохранение данных в базу данных через Django ORM
32. Добавить команду управления `python manage.py parse_products` для запуска парсера

## Этап 8: Платежные системы и безопасность
33. Установить библиотеку `pip install django-sslserver` - для HTTPS в разработке
34. Установить библиотеку `pip install django-cors-headers` - уже установлено, проверить настройки
35. Настроить HTTPS и SSL сертификаты для production
36. Интегрировать платежную систему (например, Stripe, PayPal или локальную систему)
37. Установить библиотеку `pip install cryptography` - для шифрования чувствительных данных
38. Настроить защиту от CSRF атак
39. Настроить защиту от SQL инъекций (Django ORM уже защищает)
40. Включить валидацию всех входных данных

## Этап 9: Защита персональных данных (GDPR/законодательство)
41. Установить библиотеку `pip install django-gdpr` - для соответствия GDPR (опционально)
42. Создать политику конфиденциальности
43. Настроить согласие на обработку персональных данных
44. Реализовать возможность удаления пользовательских данных
45. Настроить логирование доступа к персональным данным
46. Зашифровать чувствительные данные в базе (номера карт, адреса)

## Этап 10: Файлы конфигурации проекта
47. Создать файл `requirements.txt` со всеми зависимостями проекта
48. Настроить файл `.gitignore` для исключения venv, .env, db.sqlite3, __pycache__
49. Создать файл `README.md` с описанием проекта и инструкциями по запуску
50. Настроить файл `docker-compose.yml` для контейнеризации (опционально)

## Этап 11: Frontend (React)
51. Установить библиотеку `npm install axios` - для HTTP запросов к API
52. Установить библиотеку `npm install react-router-dom` - для маршрутизации
53. Установить библиотеку `npm install @tanstack/react-query` - для управления состоянием API запросов
54. Установить библиотеку `npm install react-hook-form` - для работы с формами
55. Установить библиотеку `npm install zod` - для валидации форм
56. Установить библиотеку `npm install zustand` или `npm install redux` - для глобального состояния
57. Настроить interceptors для axios для автоматической отправки JWT токенов
58. Создать компоненты для страниц: Главная, Каталог, Товар, Корзина, Заказ, Профиль

## Этап 12: Дополнительные улучшения
59. Установить библиотеку `pip install Pillow` - для работы с изображениями товаров
60. Настроить загрузку и обработку изображений товаров
61. Установить библиотеку `pip install celery` - для фоновых задач (парсинг, отправка email)
62. Установить библиотеку `pip install redis` - для кеширования и Celery
63. Настроить кеширование часто запрашиваемых данных
64. Настроить отправку email уведомлений (заказы, регистрация)
65. Установить библиотеку `pip install django-storages` и `pip install boto3` - для хранения файлов в S3 (опционально)

## Этап 13: Тестирование и деплой
66. Установить библиотеку `pip install pytest-django` - для написания тестов
67. Написать unit-тесты для моделей и API
68. Настроить CI/CD pipeline (GitHub Actions, GitLab CI)
69. Настроить production settings (DEBUG=False, ALLOWED_HOSTS)
70. Настроить мониторинг и логирование (Sentry, логирование ошибок)
